# Magic Sketch 초보자용 개발 가이드 및 매뉴얼 (v2.0)

본 문서는 'Magic Sketch' 앱의 기술적 구조를 이해하고, 직접 수정 및 확장하려는 입문자를 위한 상세 가이드입니다. v2.0에서는 '바이브 코딩'을 위한 필수 지식과 실전 팁이 추가되었습니다.

---

## 1. 프로젝트 개요
'Magic Sketch'는 아이들의 상상력을 현실로 만들어주는 **AI 기반 색칠공부 생성기**입니다.
*   **프론트엔드**: React + TypeScript + Tailwind CSS
*   **백엔드**: Firebase (Auth, Firestore)
*   **AI 엔진**: Google Gemini API

---

## 2. 필수 준비물 (Prerequisites)
이 프로젝트를 직접 수정하려면 다음 계정들이 필요합니다.
1.  **Google 계정**: AI Studio 및 Firebase 설정에 사용됩니다.
2.  **API Key**: Gemini 모델을 호출하기 위한 열쇠입니다. (AI Studio 설정에서 자동 관리됨)

---

## 3. 단계별 개발 가이드

### 1단계: 코드 구조 이해하기 (어디에 무엇이 있나요?)
*   `src/App.tsx`: 대부분의 화면 디자인과 로직(AI 호출, 데이터 저장)이 들어있는 **심장**입니다.
*   `src/lib/firebase.ts`: Firebase 데이터베이스와 연결하는 **통로**입니다.
*   `firebase-blueprint.json`: 데이터베이스가 어떤 모양으로 생겼는지 정의한 **설계도**입니다.
*   `DOC/`: 현재 읽고 계신 매뉴얼과 로드맵이 들어있는 **문서 보관실**입니다.

### 2단계: 화면 수정하기 (디자인 바꾸기)
가장 쉬운 수정은 글자나 색상을 바꾸는 것입니다.
*   **텍스트 변경**: `src/App.tsx` 파일에서 `Ctrl+F`를 눌러 "Magic Sketch"를 찾은 뒤 원하는 이름으로 고쳐보세요.
*   **색상 변경**: Tailwind CSS 클래스를 사용합니다. 예를 들어 `bg-[#6C5DD3]`는 보라색 배경입니다. 이를 `bg-[#FF0000]`로 바꾸면 빨간색이 됩니다.

### 3단계: AI 프롬프트 튜닝하기
`handleGenerate` 함수 안에는 AI에게 내리는 명령(Prompt)이 있습니다.
*   `"pure white background, no shading"` 같은 문구를 추가하거나 수정하여 생성되는 그림의 스타일을 바꿀 수 있습니다.

---

## 4. 데이터베이스 관리 (Firestore)
우리는 **Subcollection**이라는 구조를 사용합니다.
*   `books`: 도안의 제목, 날짜, 썸네일 정보만 저장 (가볍게!)
*   `books/{bookID}/images`: 실제 고용량 이미지 4개를 각각 따로 저장 (용량 초과 방지!)

---

## 5. 버전 관리 규칙
수정을 마칠 때마다 다음을 수행하세요:
1.  `metadata.json`의 `name` 뒤에 버전을 업데이트합니다 (예: v1 -> v2).
2.  `DOC/version_history.md`에 변경된 내용을 한 줄 기록합니다.
3.  매뉴얼(`DOC/guide_manual_vX.md`)을 복사하여 새로운 버전으로 만듭니다.

## 별첨 1: 바이브 코딩 필수 지식 20종 (입문용)
1. **바이브 코딩 (Vibe Coding)**: 자연어로 대화하듯 코딩하는 직관적인 개발 방식.
2. **LLM (거대언어모델)**: 코딩을 돕는 눈에 보이지 않는 똑똑한 AI 조수 (Gemini 등).
3. **프롬프트 (Prompt)**: AI에게 내리는 '구체적인 명령' 그 자체.
4. **컨텍스트 (Context)**: AI가 이전 대화를 기억하고 이해하는 범위.
5. **토큰 (Token)**: AI가 텍스트를 처리하는 최소 단위 (비용과 속도에 영향).
6. **기술 스택 (Tech Stack)**: 우리 앱을 만드는 재료 모음 (React, Firebase 등).
7. **상태 관리 (State)**: 앱 안에서 데이터가 바뀌면 화면도 자동으로 바뀌게 하는 기술.
8. **배포 (Deployment)**: 내가 만든 코드를 실제 인터넷 주소로 올리는 것.
9. **저장소 (Repository)**: 코드가 보관된 클라우드 창고.
10. **UI/UX**: 앱의 보기 좋은 모습(UI)과 사용하기 편한 흐름(UX).
11. **반응형 디자인**: 폰, 태블릿, PC 어디서나 화면이 예쁘게 맞춰지는 것.
12. **디버깅 (Debugging)**: 벌레를 잡듯 코드 속 오류를 찾아 해결하는 것.
13. **리팩토링**: 결과는 똑같지만 코드 내용을 더 깨끗하고 튼튼하게 고치는 것.
14. **API**: 서로 다른 앱이나 서버가 대화하기 위한 약속된 방식.
15. **환경변수 (.env)**: API 키처럼 남들에게 보여주면 안 되는 '비밀 설정값'.
16. **커밋 (Commit)**: "여기까지 수정했어!"라고 기록을 저장하는 행위.
17. **병합 (Merge)**: 따로 만든 두 코드를 하나로 합치는 것.
18. **반복 개진 (Iteration)**: 조금씩 고쳐가며 앱을 완성해 나가는 과정.
19. **보일러플레이트**: 요리 키트처럼 기본이 다 준비된 코드 템플릿.
20. **하드코딩**: 나중에 바꾸기 힘들게 코드 안에 값을 직접 적어버리는 것.

## 별첨 2: 실전 수정 요청 프롬프트 예시 20종
1. "전체 배경색을 아이들이 좋아할 파스텔톤 보라색으로 바꿔줘."
2. "로그인 버튼 옆에 구글 로고를 작고 귀엽게 넣어줘."
3. "이미지 생성 중에 반짝이는 별들이 춤추는 애니메이션을 보여줘."
4. "PDF 파일명을 '아이이름_주제_날짜.pdf'로 다운로드 되게 해줘."
5. "갤러리 카드에 마우스를 올리면 카드가 살짝 커지면서 둥둥 뜨는 효과를 줘."
6. "아이 이름을 입력하지 않으면 '이름을 적어주세요!'라고 친절하게 알려줘."
7. "최근에 만든 도안 3개를 메인 홈 화면 맨 아래에 보여줘."
8. "폰트를 전체적으로 더 동글동글하고 귀여운 느낌으로 통일해줘."
9. "주제 입력창 옆에 '주제 추천' 버튼을 만들어서 랜덤 아이디어를 넣어줘."
10. "PDF 마지막 페이지에 '참 잘했어요!'라는 축하 페이지를 한 장 추가해줘."

12. "저장 용량을 아끼기 위해 갤러리 목록에는 첫 번째 그림만 보여줘."

16. "내 폰에서 버튼이 너무 작아. 손가락으로 누르기 편하게 더 크게 만들어줘."
17. "그림 4장을 바둑판 모양(2x2)으로 예쁘게 정렬해줘."
18. "PDF를 받았을 때 한글이 하나도 안 깨지게 이미지 방식으로 그려줘."
19. "갤러리를 불러올 때 로딩 바 대신 귀여운 로봇 아이콘이 회전하게 해줘."
20. "아이의 나이에 따라 도안의 복잡도를 조절할 수 있는 슬라이더를 추가해줘."

## 별첨 3: 흔한 오류 대처 요령 20종 (치트시트)
1. **에러 발생 시**: 무서워 말고 에러 메시지를 복사해서 AI에게 그대로 붙여넣으세요.
2. **화면이 안 변할 때**: 브라우저 새로고침(F5)을 하거나 AI에게 다시 적용해달라고 하세요.
3. **무한 로딩**: 인터넷 연결을 확인하거나 Firebase 설정이 누락됐는지 물어보세요.
4. **디자인이 망가졌을 때**: "방금 전 디자인이 이상해졌어. 이전 단계로 되돌려줘"라고 말하세요.
5. **기능이 작동 안 할 때**: "방금 수정한 버튼이 안 눌려. 코드를 다시 검토해줘"라고 하세요.
6. **AI가 딴소리할 때**: "내 말은 그게 아니라 A를 하고 싶다는 거야"라고 더 구체적으로 말하세요.
7. **파일을 못 찾을 때**: "파일 경로가 바뀐 것 같아. 현재 폴더 구조를 확인해줘"라고 하세요.
8. **권한 에러 (Firestore)**: "Firebase 보안 규칙(rules)에서 내 이메일 권한이 있는지 봐줘"라고 하세요.
9. **이미지 용량 초과**: "이미지 데이터가 너무 커서 안 올라가. Subcollection으로 나눠줘"라고 하세요.
10. **한글 깨짐**: "한글 폰트가 깨져. Canvas 렌더링이나 이미지 방식으로 해결해줘"라고 하세요.
11. **중복 저장**: "한 번 눌렀는데 두 번 저장돼. useEffect의 무한 루프를 확인해줘"라고 하세요.
12. **버튼이 겹칠 때**: "모바일 화면에서 버튼들이 겹쳐 보여. 레이아웃을 다시 잡아줘"라고 하세요.
13. **API 한도 초과**: "Gemini API 무료 한도가 끝난 것 같아. 다른 모델을 써야 할까?"라고 물으세요.
14. **PDF 다운로드 안 됨**: "jspdf 라이브러리 설정이 잘못됐는지 검토해줘"라고 하세요.
15. **로그인 창 안 뜸**: "Google Auth 팝업 창이 차단됐는지 확인하는 코드를 넣어줘"라고 하세요.
16. **삭제했는데 남아있을 때**: "삭제 후 상태(State) 업데이트가 안 됐어. 갤러리 목록을 새로고침해줘"라고 하세요.
17. **엉뚱한 그림 생성**: "AI 프롬프트가 너무 모호해. 더 명확한 단어로 명령어를 수정해줘"라고 하세요.
18. **앱이 너무 무거울 때**: "이미지 로딩 속도가 느려. 지연 로딩(Lazy Loading)을 적용해줘"라고 하세요.
19. **설명이 어려울 때**: "이 부분이 너무 복잡해. 초등학생도 이해하게 아주 쉽게 설명해줘"라고 하세요.
20. **최종 점검**: 모든 수정 후 "전체 앱의 고칠 점이 없는지 마지막으로 린트(Lint) 체크해줘"라고 요청하세요.
