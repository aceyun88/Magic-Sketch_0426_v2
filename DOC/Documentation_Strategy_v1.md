# Magic Sketch 문서화 관리 전략 (v1.0)

본 문서는 프로젝트의 지속 가능성과 협업 효율을 위한 표준 문서 관리 방법론을 정의합니다.

## 1. 문서 체계 (Standard Folders)

- **README.md**: 프로젝트 루트에 위치. 가장 먼저 읽어야 할 요약본.
- **PRD (Product Requirement Document)**: 제품의 기획 의도와 비전, 수익화 방향성 정의.
- **Guide Manual**: 개발 및 사용자를 위한 실무 지침서 (바이브 코딩 팁 포함).
- **Version History**: 모든 변경 이력을 기록하여 추적 가능성 확보.

## 2. 관리 방법론: Documentation as Code

1. **가독성 중심**: 모든 문서는 Markdown(.md) 형식으로 작성합니다.
2. **코드와의 동기화**: 기능 코드가 변할 때 관련 파일(PRD의 기능 명세 또는 Manual의 사용법)을 즉시 수정합니다.
3. **이력 보존**: 파일명에 버전을 붙이는 방식은 '스냅샷' 용도로 사용하고, 최신본은 항상 `/DOC` 내의 최상위 버전을 참조하도록 합니다.

## 3. 문서 간 상호 참조 관계

- **전략(PRD)** -> **구현(Code)** -> **교육(Manual)** 순으로 연쇄적인 업데이트를 권장합니다.
